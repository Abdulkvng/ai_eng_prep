# 00 - Interview Map

## What this role is really testing

AI Enablement is about helping engineers use AI better.

You are not only building models. You are building the systems, tools, workflows, docs, and experiments that make AI useful inside a company.

## The main interview buckets

### 1. Can you build AI systems?

They may ask you to design an internal AI assistant, an AI workflow tool, or an LLM usage platform.

You should mention:

- LLM provider integration
- prompt design
- retrieval
- tools
- evals
- guardrails
- observability
- cost control

### 2. Can you explain tradeoffs?

AI systems are tradeoff-heavy.

Common tradeoffs:

- cheaper model vs stronger model
- lower latency vs higher quality
- flexible output vs structured output
- agent autonomy vs safety
- more context vs higher token cost

### 3. Can you evaluate if AI works?

Do not just say "it seems good."

Say you would measure:

- task success rate
- hallucination rate
- latency
- cost per request
- user adoption
- human override rate
- answer helpfulness

### 4. Can you communicate clearly?

This role is cross-functional. You may work with engineers, product, risk, support, data, and leadership.

Good communication means explaining technical decisions in simple language.
